# ChessIt ONNX training pipeline

This folder contains the Python pipeline for creating the `chessit_ai.onnx` model used by the ChessIt C++ game.

## Model contract

The exported ONNX model has the following interface:

| Name | Direction | Shape | Type | Meaning |
|---|---:|---:|---:|---|
| `board` | input | `[batch, 13, 8, 8]` | `float32` | Board tensor planes. |
| `move_logits` | output | `[batch, 4096]` | `float32` | Score for each `from_square * 64 + to_square` move. |
| `value` | output | `[batch, 1]` | `float32` | Position value from the side-to-move perspective. |

Board planes:

| Plane | Content |
|---:|---|
| 0 | White pawns |
| 1 | White knights |
| 2 | White bishops |
| 3 | White rooks |
| 4 | White queen |
| 5 | White king |
| 6 | Black pawns |
| 7 | Black knights |
| 8 | Black bishops |
| 9 | Black rooks |
| 10 | Black queen |
| 11 | Black king |
| 12 | Side to move: `+1.0` for white, `-1.0` for black |

Squares use the `python-chess` convention: `a1 = 0`, `b1 = 1`, ..., `h8 = 63`.

```powershell

python training/train_chessit_model.py --pgn training/data/games.pgn --output models/chessit_ai.onnx
```

Linux/macOS:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r training/requirements.txt
python training/train_chessit_model.py --pgn training/data/games.pgn --output models/chessit_ai.onnx
```

## Useful options

Train with fewer positions for a fast test:

```bash
python training/train_chessit_model.py --pgn training/data/games.pgn --output models/chessit_ai.onnx --max-positions 1000 --epochs 1
```

Validate the ONNX export path without using PGN data:

```bash
python training/train_chessit_model.py --skip-training --output models/chessit_ai.onnx
```

## Using the model in the game

For runtime use, copy the trained model to:

```text
bin/chessit_ai.onnx
```

The Makefile now builds `bin/chessit_3d.exe` by default, so the executable, DLLs, and ONNX model stay together.

The game should always filter `move_logits` through the legal moves generated by the C++ chess rules. The neural model scores moves; it should not be allowed to make illegal moves directly.
